# Security Policy

## Supported Versions

- Before `v0.1.0`, security fixes are applied to the active development branch.
- After `v0.1.0`, the latest `v0.1.x` release receives security fixes unless a later policy supersedes it.

## Private Reporting Mechanism

Use GitHub Private Vulnerability Reporting. Do not report vulnerabilities through public issues.

## Response Expectations

- **Acknowledgment:** Within 5 business days.
- **Initial triage:** Within 10 business days.
- **Status updates:** Every 14 days while the investigation remains open.

## Responsible Disclosure

Please do not disclose the issue publicly before a fix or coordinated disclosure date.

## Scope

- **In scope:** Agentic Blender code and packaged resources.
- **Out of scope:** Blender itself and vulnerabilities exclusively in third-party dependencies, though dependency reports may still be forwarded or mitigated.

## No Guarantee of a Fixed Resolution Date

Resolution timing depends on severity, reproducibility, and upstream dependencies.
